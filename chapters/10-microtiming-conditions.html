Microtiming & conditional trigs

Two features that lift the Octatrack sequencer above the step-quantised grid — microtiming pushes trigs off the grid; conditional trigs make trigs probabilistic or context-dependent.

Microtiming

By default, every trig fires exactly on its step's beat division. Microtiming offsets a single trig forward or backward in time by a fraction of a step — adding push, pull, swing, or human inconsistency to a single hit.

To set microtiming: hold the trig key, then press the LEFT or RIGHT arrow key. Each press shifts the trig by 1/23rd of a step. The OT shows the offset on the LCD as a small numeric value (negative = earlier, positive = later).

Microtiming range is roughly ±half a step. Pushing further wraps around (a "+12" trig on step 5 fires earlier than the next step's natural time, but not earlier than step 5 itself).

Practical uses

  • Swing — add positive microtiming to every other step. Common values: +5 to +10 for subtle swing, +12 to +18 for broken-beat feel.
  • Drag and push — pull the snare back ~3 ticks for a laid-back feel; push the hi-hat forward 2–4 for urgency.
  • Humanisation — randomise small amounts (±2 to ±5) across an otherwise rigid pattern.
  • Polyrhythmic phasing — combined with per-track pattern lengths, microtiming creates complex evolving textures.

Conditional trigs

A conditional trig only fires if a condition is met. Conditions are set per-trig via the LEVEL knob while a trig is held. As you turn LEVEL with a trig held, the LCD displays the active condition.

The OS 1.40A condition list, in order:

ConditionMeaning
FILLFires only when fill mode is active
FILL (inverted)Fires only when fill mode is NOT active
PREFires if the previous trig on this track fired
PRE (inverted)Fires if the previous trig did NOT fire
NEIFires if the same step on the neighbouring (lower-numbered) track fired
NEI (inverted)Fires if the neighbouring track step did NOT fire
1STFires only on the first cycle of the pattern
1ST (inverted)Fires on every cycle EXCEPT the first
X%Probability — fires X% of the time (10, 20, ... 90%)
A:BFires on the A-th occurrence out of every B (e.g. 1:2 = every other; 2:3 = the second of every three)

Using conditions

The most useful conditions in practice:

  • FILL — keep fills off the main pattern; they only appear when you trigger fill mode.
  • 50% — the workhorse. Add probabilistic hi-hat flutters, ghost snare hits, occasional accents.
  • 1:2 — alternates between hits on even/odd cycles, useful for variations.
  • PRE — chains. A trig with PRE only fires if the previous step's trig also fired — useful for ratchet-style repeats.
  • NEI — couples tracks. A trig on T2 with condition NEI only fires if T1's same-step trig fired. Useful for accent layers tied to a kick.

Recipe — probability cascade for hi-hats

On a 16-step pattern, place a hi-hat trig on every step. Set conditions:

  • Steps 1, 5, 9, 13: no condition (always fire — the downbeats)
  • Steps 3, 7, 11, 15: 80% (steady upbeats with occasional dropouts)
  • Steps 2, 6, 10, 14: 50% (off-beats that come and go)
  • Steps 4, 8, 12, 16: 30% (rare flourishes)

The hi-hat now feels alive — different on every cycle, but the core groove (1, 5, 9, 13) is rock-solid.

Fill mode

Fill mode is a momentary switch that activates all FILL-conditional trigs. Activate with FUNCTION + YES (toggle), or hold FUNCTION + YES for momentary (release ends the fill).

The classic use: design the main groove without fills, then add a parallel layer of FILL-conditional trigs that only fire when you momentarily activate fill mode. Ride the fill switch during a performance to introduce variation without changing patterns.

Inverted-FILL trigs fire only when fill is off — useful for "drop the kick during the fill" effects. Combine FILL and inverted-FILL on the same pattern for clean swap-outs.

Combining conditions

Each trig has one condition. To stack effects, use multiple trigs on adjacent steps with different conditions, or use per-track pattern lengths to create longer-evolving sequences.

The combination of microtiming + conditional trigs + parameter locks is what makes the Octatrack's sequencer feel generative rather than mechanical. Two patterns, one part, careful use of conditions: indistinguishable from improvisation in many contexts.

Hidden · NEI condition pitfall

The NEI condition references the previous track (e.g. T2's NEI condition checks T1; T3 checks T2). It does NOT chain across all tracks. T1's NEI references T8, wrapping around. A common mistake is expecting NEI on T5 to depend on T1; it depends on T4. Plan track ordering accordingly.
