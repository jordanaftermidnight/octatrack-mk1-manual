Chapter 7

Audio Pool, Static, Flex

How samples physically exist on the Octatrack — where they live on disk, when they load into RAM, and the difference between Static and Flex memory. Get this right and the rest of the machine makes sense.

Audio Pool

Every sample available to a project lives in the Audio Pool — a folder on the CF card, scoped to the current set. The Audio Pool is just files: WAV or AIFF, mono or stereo, 16-bit or 24-bit, at any sample rate up to 96 kHz. The Octatrack will resample on load if needed; for cleanest playback, normalise to 44.1 or 48 kHz beforehand.

Samples are referenced from the project, not embedded in it. Deleting a sample from the Audio Pool while a project references it produces an error message (and a silent track) on next load. The Octatrack does not "save with project" the way a DAW does — your project file is small (a few KB); the audio lives separately in the Pool.

Maximum samples per Audio Pool: 999. Practical limit: card speed and your patience while it indexes.

Static slots vs Flex slots

A project holds two parallel sample-slot lists: 128 Static slots and 128 Flex slots. A track loaded with a Static machine plays from the Static list; a Flex track plays from the Flex list. The two lists are independent — slot 1 in Static and slot 1 in Flex are different samples (or empty).

The slot list is a project-level construct. It maps a slot number to a file in the Audio Pool, plus per-slot trim points (start/end loop points), gain, and BPM info.

The fundamental difference: where the audio lives

Static

Audio for a Static slot is streamed from the CF card on demand. The Octatrack reads the file as the playhead moves through it. RAM usage is essentially zero. Disk-streaming has constraints:

  • Cannot reverse-play (no random-access fast enough)
  • Cannot pitch significantly down (slows playback past disk-read speed)
  • Limited to the file's natural length and direction

Best for: long files (full songs, stems, loops over a few seconds), backing tracks, anything where the file plays linearly once at a time.

Flex

Audio for a Flex slot is loaded into RAM on project load. RAM usage is the file size — a 30-second 44.1 kHz stereo WAV is about 5 MB. With the OT's RAM split between Flex and other pools (configured via Memory Config), you can hold roughly 80 MB of Flex samples by default.

  • Reverse-play, pitch up/down, scrubbing — all instant
  • Slice playback — instant jump to slice points
  • Real-time time-stretch (TIMESTRETCH page)

Best for: drum hits, one-shots, short loops, anything that needs playback flexibility, anything you want to slice.

Memory configuration

The Octatrack's RAM is partitioned at the project level via PROJECT → MEMORY CONFIG. The total available is fixed; you choose how to split it between:

  • Flex sample memory — your Flex slots
  • Recorder memory — capacity of the eight recorder buffers (one per audio track), in mono seconds, in stereo seconds, and at 24-bit or 16-bit
  • Reserve — what's left for the OS

The default split is balanced for general use. If you need long recording buffers (multi-bar live capture), reduce Flex memory and increase recorder memory. If you need many short Flex samples, reverse the trade.

Note · Memory Config requires reboot

Changing Memory Config doesn't take effect until the project is saved and reloaded. The OT prompts you for this. Don't change Memory Config mid-performance.

When does loading happen?

On project load, the OT reads the slot lists and:

  • Scans the Audio Pool for each referenced file
  • Loads every Flex slot's audio into RAM
  • Indexes each Static slot's file (does not load the audio)

A project with 50 small Flex samples loads in seconds. A project with a hundred multi-megabyte Flex samples can take 30–60 seconds. The Card Status LED on the front panel indicates load activity.

Practical workflow

Recipe — choosing Static or Flex

Quick decision tree:

  1. Sample longer than 30 seconds and plays linearly? → Static
  2. Need to slice it, reverse it, time-stretch it, or pitch it significantly? → Flex
  3. It's a drum hit, one-shot, or short loop? → Flex
  4. It's a backing track, full-length stem, or DJ source? → Static
  5. Both? Load it as both — Static and Flex slots are independent, and many performances assign the same audio file to both lists for different uses.

The Sample Manager

PROJECT → SAMPLE MANAGER opens the unified file browser for the Audio Pool. From here you assign samples to slots, audition them, set per-slot trim points, view file metadata, and clean up unused samples.

The Sample Manager also manages the file-system view of the Audio Pool: rename, move between sub-folders, delete, batch-load. For detailed Audio Pool management workflows, see chapter 27.
