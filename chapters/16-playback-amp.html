PLAYBACK & AMP pages

The first two parameter pages: PLAYBACK shapes how the sample plays; AMP shapes its envelope and level.

PLAYBACK page

The PLAYBACK page exposes six parameters. For Static and Flex machines they're broadly equivalent, with Flex offering a wider range on several:

KnobParamRangeFunction
ASLOT0–127Sample slot for this track
BPTCH±64 (Flex), ±24 (Static)Pitch in semitones
CSTRT0–127 / sliceSample start point (or slice number)
DLEN0–127Length of playback from start point
ERATE±64Playback rate (negative = reverse on Flex)
FLOOPOFF / ONLoop the playback within the start/length window

Pitch and rate together

Pitch and rate are independent on Flex tracks (with TIMESTRETCH on) but coupled on Static. For Static, changing pitch changes rate (you're slowing the file down, which lowers pitch). For Flex with TIMESTRETCH off, same coupling. With TIMESTRETCH on, you can pitch without time-shifting, or vice versa.

Sample start, length, loop

These three define the playback window within the sample:

  • STRT = 0 plays from the file's beginning
  • LEN = max plays the full length of the file from STRT
  • LOOP ON loops the start/length window indefinitely (until the next trig or until the AMP envelope releases)

P-locking STRT per step is the technique behind glitch and stutter effects: 16 trigs on a sample, each with a different STRT lock, plays 16 different fragments of the file as a sliced loop.

Slice mode

When a Flex sample has a slice grid defined (chapter 30), STRT becomes a slice number selector instead of a position. P-locking STRT then chooses which slice each step plays.

SLICES trig mode (chapter 11) automatically locks STRT to the trig number's slice — but you can also do it manually via p-locks in TRACKS mode.

AMP page

The AMP page shapes the track's amplitude envelope and overall level:

KnobParamRangeFunction
AATK0–127Attack time
BHOLD0–128Hold time at full level (128 = infinite)
CREL0–128Release time
DVOL0–127Track output level
EBAL±64Stereo balance (pan)
FXVOL0–127 / ±127Crossfader-controlled volume offset (see chapter 23)

The envelope shape

The OT's amplitude envelope is AHR (Attack-Hold-Release). No sustain stage — HOLD substitutes for sustain by holding peak level for a fixed duration before releasing.

HOLD = 128 means infinite hold — the envelope reaches peak and stays there until a new trig fires (or the track is muted, or something else interrupts). This is the OT's "sustain" mode.

Relationship to sample length

The envelope and the sample's natural length are independent. A short envelope on a long sample cuts the sample short; a long envelope on a short sample lets the sample play through and then holds silence (with potential filter/FX tails) for the remaining envelope time.

XVOL — the crossfader-controlled level

XVOL is a level offset that responds to the crossfader. With XVOL set to a positive value and the fader in scene-locked positions, the track's effective volume becomes VOL + (XVOL × fader_position).

The mechanism is more deeply useful with scenes — see chapter 23 for the full crossfader integration. For now: XVOL is the parameter you scene-lock when you want the crossfader to fade a track in or out.

Note · the XVOL trap

XVOL multiplies with VOL. A track with VOL = 0 cannot be brought up by XVOL — XVOL of any value × VOL of 0 = 0. To use XVOL for "fade in", set VOL to a non-zero value first, then scene-lock XVOL across the fader. Or use scene-lock on VOL directly. This is one of the OT's most-encountered "why doesn't this work?" issues.

Practical AMP page recipes

Recipe — kick stab

ATK = 0, HOLD = 4, REL = 12. Short, percussive, no tail. Good for short-sample drums and bass stabs.

Recipe — pad swell

ATK = 30, HOLD = 0, REL = 50. Slow attack, gradual release. The sample fades in and gently fades out — useful for ambient pads from a long sample.

Recipe — sustained drone

ATK = 0, HOLD = 128, REL = 0. The envelope holds infinitely. One trig produces continuous sound until something interrupts. Use with LOOP ON on PLAYBACK for true sustain.
