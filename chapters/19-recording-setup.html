Chapter 19

Recording setup

The Octatrack samples on the fly into eight per-track recorder buffers. This chapter covers how to configure recording — sources, lengths, formats — and how the recorder interacts with the sequencer.

The recorder buffers

Every audio track has its own recorder buffer. There are eight buffers total, one per track. They share a common pool of memory (set in PROJECT → MEMORY CONFIG, see chapter 7).

Each buffer can be:

  • Triggered by REC AB / REC CD on the front panel
  • Triggered by a sequencer record-trig
  • Played back as a sample by any Static or Flex track that references it
  • Saved to the Audio Pool as a permanent sample (the "Save Recording" option)

Recording Setup pages

Two setup pages, accessed via:

  • FUNCTION + REC AB — page 1
  • FUNCTION + REC CD — page 2

The settings are per-track — each track has its own recording config.

Memory config

The recorder memory pool is split into eight buffers. Memory settings (PROJECT → MEMORY CONFIG):

  • Recorder buffer length — total memory, in seconds. Distributed equally across the 8 buffers.
  • Mono / stereo — mono buffers double their recordable length
  • 16-bit / 24-bit — 16-bit doubles the length compared to 24-bit

The default memory config gives roughly 30 seconds per buffer at 16-bit stereo. Bumping up to 60+ seconds requires either reducing Flex memory or going mono/16-bit.

Triggering recording

Several ways to start recording:

  • REC AB — record AB pair into the active track's buffer (record-quick mode), or hold a TRACK button to record into that specific track
  • A record-trig in the sequencer — placed on a step like a normal trig, but on a "record track" view
  • External MIDI message (CC remote-trigger of recording)

Quantised recording

With QREC set to a non-zero value, the recorder waits until the next quantise boundary before actually starting. Press REC AB on beat 2.5; with QREC = 1/4 bar, recording begins at beat 3.

Useful for: capturing aligned-to-beat material live, where you can't time button presses precisely. The recorder does the quantising for you.

Length presets vs free record

RLEN sets a fixed length in steps. The recorder captures exactly that much, then stops. For a 16-step pattern at 1/16 time scale, RLEN = 16 captures one bar.

FREE mode disables RLEN; you stop recording manually. The recorder captures until you press the same trigger again, or until the buffer is full. Useful when you don't know the length in advance.

Note · the buffer is volatile

Recorder buffers are in RAM. They're not saved on power-off, and they're not saved with the project unless you explicitly save them via "Save Recording" (PROJECT → CARD TOOLS). For live performance this is fine — buffers regenerate as you sample. For anything you want to keep, save explicitly.

Playing back recordings

Recorder buffers are accessible as samples. On a Flex or Static machine, you can assign a recorder buffer as the slot source (via the Setup page). Recording on T1 and playing the result back through a Flex machine on T1 means T1 captures and replays in the same track — the basis for live looping.

Pickup machines (chapter 15) wrap this entire workflow into a single machine type. For workflows where you want explicit control over capture timing and slot assignment, the manual recorder setup gives finer control.
