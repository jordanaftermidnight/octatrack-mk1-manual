Appendix B

Troubleshooting

Common problems and their fixes. Plus the early startup menu — the OT's diagnostic and recovery shell, accessed by holding trig keys at boot.

"My changes disappeared"

By far the most common problem. Almost always: an unsaved part.

  • Did you save the part? FUNCTION + YES on the part page.
  • Did you save the pattern? FUNCTION + YES on the pattern page.
  • Did you save the project? PROJECT → SAVE.

Patterns, parts, and projects each have their own save state. All three need committing to disk before power-off, or the changes are lost. The OT does not auto-save.

"My CF card is corrupted"

Symptoms: project won't load, samples missing, weird errors at boot. Usually caused by power-cycling during a write operation.

Steps:

  1. Power off the OT cleanly (don't yank power)
  2. Remove the CF card and insert it into a card reader on a computer
  3. Run the OS-level disk repair (chkdsk on Windows, Disk Utility on macOS, fsck on Linux)
  4. If repair succeeds, reinsert and reboot the OT
  5. If repair fails, reformat (FAT32) and restore from backup

"I can't get sound out of a Flex track"

Diagnostic:

  1. Is a sample assigned? Check the SLOT parameter.
  2. Is the track muted? Check track LED on the trig row.
  3. Is the AMP VOL > 0?
  4. Is the AMP envelope set sensibly? (ATK = 0, HOLD > 0, REL > 0)
  5. Are there trigs on the pattern?
  6. Is the track routed to MAIN? (Mixer page)
  7. Is the FX2 reverb mix at 0 with reverb dry-only? Sometimes you've cranked dry/wet to wet-only.
  8. Is the crossfader scene-locked to mute the track?

"My Pickup track won't start recording"

  • Is the trig a one-shot (YES + TRIG)? Plain trigs don't trigger Pickup state changes.
  • Is the source set correctly? Check SETUP → SRC.
  • Is there an input signal? Check the AB / CD level meters at the top.
  • Is RTRIG enabled? Auto-retrigger sometimes interferes.
  • Is the buffer already full from a previous capture? Clear with FUNCTION + the one-shot trig.

"Crossfader doesn't do anything"

  • Are scenes assigned to A and B? Check SCENE A / SCENE B button LEDs.
  • Are any parameters scene-locked? If neither scene has any locked parameters, the fader has nothing to morph.
  • Are A and B assigned to the same scene? They'd produce identical output across the fader range.
  • Is XVOL the parameter you locked, but VOL is at 0? See chapter 16's XVOL trap.

"Audio is glitchy / dropouts"

  • Check Memory Config — too many Static streams from a slow card may cause dropouts. Convert some to Flex.
  • CF card speed: slow cards (sub 120x) struggle with multiple Static tracks. Fast cards (300x+) are recommended.
  • Reformat the card if you suspect file-system fragmentation.
  • Power supply: undervoltage causes intermittent issues. Use the official Elektron PSU, not a generic 12V DC.

"Tempo drifts when slaved"

  • Long MIDI THRU chains accumulate jitter. Use a re-clocking splitter for chains over 3 devices.
  • Some master devices send unstable clock when their internal sequencers are doing other work. Check by slaving the OT to a known-good clock source (e.g. a different drum machine).
  • Two Elektron devices benefit from TurboMIDI for tighter sync. Verify both are connected by a single MIDI cable directly.

The Early Startup Menu

Hold TRIG 1, TRIG 2, TRIG 3, or TRIG 4 while powering on the OT. Each opens a different diagnostic / recovery shell.

TRIG 1 — Test mode

Hardware test menu. Tests the LCD, all keys, all knobs, MIDI ports, audio I/O, CF card. Useful when diagnosing physical failures (a dead key, a noisy knob, a misbehaving input). Each test cycles through the relevant hardware; press keys / turn knobs / etc., and the screen confirms whether the OT detects them.

TRIG 2 — OS upgrade

Boots into a minimal mode that can read OS .syx files from the CF card. Place the firmware file on the card root, hold TRIG 2 at boot, follow on-screen prompts. Used for OS upgrades when SysEx-over-USB isn't working.

TRIG 3 — File system tools

Format CF card, partition diagnostics, file-system repair from within the OT. Useful when the card is misbehaving but you don't have a card reader to hand.

TRIG 4 — Boot from CF in disk mode

Mounts the OT's CF card as a USB mass-storage device. Plug a USB cable from OT to computer, hold TRIG 4, and the card appears as a removable drive. Used for backing up samples, copying projects, bulk file management without a separate card reader.

Warning · Disk Mode requires unmount

Always safely eject / unmount the OT from the computer's file system before disconnecting USB or rebooting. Yanking the cable mid-write can corrupt the card. The OT does not enforce this on its end — the safety check happens on the computer.

Battery replacement

The OT MK1 has an internal CR2032 battery for the real-time clock. Lifespan: 5–8 years. When low, the OT loses date/time settings between boots (project timestamps wrong, etc.). The machine still functions but RTC is unreliable.

Replacement requires opening the unit. Reasonably accessible if you're comfortable with screwdrivers; consult Elektron's service documentation for procedure. The battery is not user-serviceable in any official sense — replacing voids any remaining warranty.
