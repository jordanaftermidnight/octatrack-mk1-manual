Chapter 20

Audio editor

The Audio Editor is where you trim, loop, slice, and analyse samples. Two main pages: TRIM for boundary editing and SLICE for slice-grid creation.

Accessing the editor

From the Sample Manager (PROJECT → SAMPLE MANAGER), select a sample and press YES. The editor opens with the sample's waveform displayed on the LCD.

Or from a track: hold FUNCTION + BANK (EDIT) with the active track selected and a Flex/Static machine loaded. This opens the editor for the track's current sample.

TRIM page

TRIM lets you set the sample's start and end points (the bounds of playback) and loop points (for sustained playback).

KnobFunction
ATRIM IN — start point of the sample
BTRIM OUT — end point of the sample
CLOOP IN — loop start point
DLOOP OUT — loop end point
EZoom level
FCursor position (for fine-tuning)

Why trim points matter

Trim points are saved with the slot list, not with the sample file itself. So one audio file can have multiple slots pointing to it with different trim/loop settings. The audio file is shared; the playback bounds are slot-specific.

Practical use: a single 30-second loop file can be referenced by 16 different Flex slots, each trimmed to a different 2-second section. One file, 16 different one-bar loops accessible per slot.

Auto-trim

The TRIM page menu offers auto-trim functions:

  • Auto trim — find silence at start/end and trim to non-silent boundaries
  • Auto loop — find a clean loop point in the sample (zero-crossings, period analysis)
  • Normalize — boost the sample's level to maximum without clipping

SLICE page

Slicing divides a sample into up to 64 named regions, each individually addressable as a "slice number" via the STRT parameter (chapter 16) or via SLICES trig mode (chapter 11).

Creating slices

From the SLICE page menu:

  • Auto-slice (grid) — divide the sample into N equal regions (4, 8, 16, 32, 64). Most common for drum loops.
  • Auto-slice (transients) — analyse the audio and place slice boundaries at detected transients. Threshold adjustable.
  • Manual slice — place slice points by ear with the cursor

Editing slices

  • Adjust slice boundaries with the cursor + knob A
  • Each slice's individual length can be tweaked
  • Delete unwanted slices or merge adjacent ones
  • Save the slice grid back to the slot

Playing slices

Once sliced, a sample's slices can be played by:

  • SLICES trig mode — each trig key plays a different slice
  • STRT p-locks — set per step in TRACKS mode for sequenced slicing
  • LFO-modulated STRT — LFO cycles through slice numbers for random or rhythmic slicing

Sample chain workflow

A "sample chain" is multiple short samples concatenated into one file, then sliced. Useful for getting more sounds into a single track with one slot.

Build a chain externally (in a DAW) — one drum hit per equal time-segment, e.g. eight 0.5-second segments forming a 4-second file. Import the chain into a Flex slot. Auto-slice 8 ways. Now each trig in SLICES mode plays a different drum hit, all from one slot, one sample.

Resampling

Resampling = recording the OT's own output back into a recorder buffer, then loading that buffer as a new sample. This is how you "freeze" a complex performance into a single sample, freeing up tracks.

Steps:

  1. Set up a track with a Flex machine and the recorder buffer of another track as its source (via SRC3 setting in Recording Setup)
  2. Configure recording with QREC for clean alignment
  3. Trigger recording while the source is playing
  4. Save the resulting buffer to the Audio Pool
  5. Load the saved sample as a new Flex slot — replacing the original tracks

Resampling is essential for live performance: a complex four- track drum arrangement can be flattened to one Flex slot, freeing three tracks for other use.

Tip · zero-crossing for clean loops

Loop points work best at zero-crossings — points where the waveform crosses the centre line. The OT's loop-finder tries to identify these automatically, but for tricky material you may need to manually nudge to a zero-crossing. Click + cursor near the suspected point; zoom in; find the cleanest crossing within a few samples. The result is artifact-free looping.
